# Contributing to fwall

We love your input! We want to make contributing to `fwall` as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## How to Contribute

### 1. Fork the Repository
Fork the repo and create your branch from `main`.

### 2. Install Dependencies
Make sure you have [Bun](https://bun.sh/) or [npm](https://www.npmjs.com/) installed. Then run:
```bash
bun install
# or
npm install
```

### 3. Make Changes
Implement your changes. Ensure your code follows the existing style and is well-documented.

### 4. Test Your Changes
Run the tests to make sure everything is still working as expected:
```bash
bun test
# or
npm run test
```

### 5. Submit a Pull Request
Submit a PR with a clear description of what you've changed and why.

## Code of Conduct
Please be respectful and professional in all your interactions with the project.

## Bug Reports
If you find a bug, please open an issue with as much detail as possible, including:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment (browser, OS, etc.)

## Feature Requests
We're always open to new ideas! Please open an issue to discuss your proposal before starting work on it.
